# 购房决策智能助手 - UI设计文档

## 概述

本文档定义了购房决策智能助手的UI设计方案。

**设计风格**：温暖亲和 + 专业可信  
**设计原则**：
- 对话优先，AI能做所有事
- 多角色展示，突出"多智能体"
- 快捷入口，降低使用门槛
- 响应式设计，为小程序做准备

## 三、页面定位

| 页面 | 定位 | 适合场景 |
|------|------|---------|
| **对话页面** | 主入口，AI自动调度所有工具 | 不知道怎么问、复杂问题、综合咨询 |
| **计算器页面** | 快捷工具，表单直接输入 | 明确知道要算什么，想快速得到结果 |
| **市场分析页面** | 数据展示，图表可视化 | 想看走势图、区域对比 |
| **政策页面** | 政策列表，快速查阅 | 想直接看政策原文 |
| **我的页面** | 用户中心 | 历史记录、收藏、设置 |

### 页面关系

```
对话页面（AI）—— 超集，能做所有事
    ├── 调用 calc_loan → 和计算器页面一样的计算能力
    ├── 调用 query_market → 和市场分析页面一样的数据
    ├── 调用 search_policy → 和政策页面一样的内容
    └── 额外能力：多角色协作、综合分析、个性化建议

其他页面 —— 快捷入口，特定场景更高效
```

**设计要点**：
- **欢迎页面**：个性化问候语 "Hi，{昵称}"，2x2 网格快捷功能卡片
- **快捷功能卡片**：包含图标、标题、描述，点击直接发送预设问题
  - 贷款计算（Coin图标）
  - 政策解读（Document图标）
  - 市场分析（TrendCharts图标）
  - 购房咨询（QuestionFilled图标）
- **思考过程**：可折叠的思考步骤展示，显示执行时间和各角色分析进度
- **多角色折叠展示**：综合建议为主，专家分析可展开查看
- **用户消息**：蓝色气泡，右对齐，圆角设计
- **输入框**：底部固定，带发送按钮和免责提示

## 七、比赛演示重点

1. **对话页面**：展示多智能体协作，不同角色分析同一问题
